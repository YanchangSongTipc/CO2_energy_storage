# Fig01 — 充电-供热运行可行域 (Charging-Heating Feasibility Domain)

## 图中元素

| 元素 | 含义 |
|------|------|
| 蓝色填充区 | 系统在当前时刻可以安全运行的所有 (W, Q) 组合 |
| 红色实线 | HTV 储热下限决定的供热上限：Q ≤ β1·W + (Q_HTV−Q_lower)/Δt |
| 红色虚线 | HTV 储热上限决定的供热下限：Q ≥ β1·W − (Q_upper−Q_HTV)/Δt |
| 黑色圆点 | 可行域多边形顶点 |

## 物理逻辑

充电时热平衡为：

```
新储热量 = 旧储热量 + (压缩热 − 供热量) × Δt
```

- **充电功率 W 越大 → 压缩热 β1·W 越多** → 可供热能力越强（斜率 **+β1**）
- 红色实线代表在当前储热状态下，给定充电功率时能提供的**最大**供热功率
- 可行域 = 所有满足储热罐不溢出也不耗尽、供热不超设备上限的 (W, Q) 组合

## 充电过程如何供热

压缩 CO₂ 时产生大量**压缩热**。两级压缩的出口温度分别达到 ~495K 和 ~506K，这些高温 CO₂ 在进入下一级之前被中间冷却器（IC1、IC2）用**导热油**吸收。

吸收了压缩热的导热油存入**高温储热罐（HTV）**，然后有两个用途：

```
                ┌─→ 回热 (β2): 放电时加热 CO₂ 驱动透平发电
压缩热 (β1·W) ─┤
                └─→ 供热 (Q_heat): 直接供给用户取暖
```

充电过程中的热平衡：

```
Q_HTV(t) = Q_HTV(t-1) + (β1·W_C − Q_heat) × Δt
```

- `+β1·W_C`：本次充电产生的压缩热，存入 HTV
- `−Q_heat`：从 HTV 取出的热量，供给用户
- 只要 Q_HTV 不降到下限以下，就可以一边充电一边供热

## 演示参数

`b_upper_c = 8` 表示假设 HTV 当前储热较低（只有 8 MWh 可用热），使正斜率约束线切入可视范围。若用系统实际参数（Q_HTV=75 MWh），约束线远在设备供热上限之上，可行域退化为矩形。

# Fig02 — 放电-供热运行可行域 (Discharging-Heating Feasibility Domain)

## 热的去向（放电时）

放电时系统**不产生**新热，只有消耗。HTV 中的储热有两个竞争性消耗：

```
                    ┌─→ 回热 (β2·W_T): 加热 CO₂ 驱动透平 → 发电
HTV 储热 ──────────┤
                    └─→ 供热 (Q_heat): 直接供给用户
```

热平衡方程：

```
Q_HTV(t) = Q_HTV(t-1) − (β2·W_T + Q_heat) × Δt
```

两项都是**减法**——储热罐只出不进。

## 负斜率 −β2：电热竞争

约束线 `Q = −β2·W + Q₀`：

- **发电 W_T 越大** → 回热消耗 β2·W_T 越多 → 留给供热的储热越少
- **供热 Q 越大** → 留给发电的储热越少 → 发电上限越低
- **两者不可兼得最大值**

## 图中元素

| 元素 | 含义 |
|------|------|
| 红色填充 | 当前可行域（梯形/三角形，非矩形） |
| 蓝色实线 | 供热上限：Q ≤ −β2·W + Q₀（负斜率） |
| 蓝色虚线 | 供热下限 |
| INFEASIBLE 灰色框 | 右上角不可达区域——最大发电与最大供热无法同时实现 |
| 蓝色方块 "Max Power" | 零供热时的最大发电功率 |
| 红色方块 "Max Heat" | 零发电时的最大供热功率 |

## 与 Fig01 的关键区别

| | Fig01 充电 | Fig02 放电 |
|------|----------|----------|
| 斜率 | **+β1**（正） | **−β2**（负） |
| 电热关系 | 互补：充越多可供热越多 | **竞争**：发越多可供热越少 |
| 可行域形状 | 梯形向**上**扩展 | 梯形向**下**收缩 |
| 右上角 | 可达（矩形可能） | **不可达**（被切掉） |

## 演示参数

`b_upper_d = 18` 表示假设 HTV 当前只有 18 MW 可用热功率（储热较紧张），使负斜率约束在可视范围内生效，展示右上角被切掉的非矩形形状。

# Fig03 — 负荷与风电出力曲线 (Load and Wind Power Curves)

## 三条曲线

| 曲线 | 含义 | 日间特征 |
|------|------|----------|
| 红色（热负荷） | 用户取暖需求 | 早晚高（~42 MW），中午低（~28 MW） |
| 蓝色（电负荷） | 用户用电需求 | 中午和晚间两个高峰（~48 MW），凌晨低谷 |
| 绿色（风电出力） | 风电机组出力 | 凌晨高（~18 MW），中午低（接近 0） |

## 这条曲线决定 CCES-CHP 的调度指令

```
电调度 = 电负荷 − C-CHP 供电 − 风电出力
热调度 = 热负荷 − C-CHP 供热
```

- **电调度 > 0** → 电有多余（弃风/低谷）→ **充电**储能
- **电调度 < 0** → 电有缺口 → **放电**补足
- **热调度 > 0** → C-CHP 供热不足 → CCES-CHP **补充供热**

## 关键时段

| 时段 | 特征 | CCES-CHP 动作 |
|------|------|-------------|
| 0-8h（凌晨） | 电负荷低 + 风电高 → 电过剩 | 充电储电 |
| 8-11h（上午） | 风电极低 | 待机或小功率 |
| 12-15h（午高峰） | 电负荷峰值 | 放电发电 |
| 19-21h（晚高峰） | 电热双高峰 | 放电+高峰供热 |

# Fig07 — 双 SOC 变化曲线 (Dual SOC Variation)

Fig07 展示 CCES-CHP 两个**储能状态量**在 24 小时内的联动演化。

## 双 SOC 模型

| SOC | 物理含义 | 存储介质 | 单位 |
|-----|---------|---------|------|
| 热 SOC（红色） | 高温储热罐 HTV 的储热百分比 | 导热油 | %（0=耗尽，100=满） |
| 气 SOC（蓝色） | CO₂ 高压储罐 HPT 的压力百分比 | CO₂ 工质 | %（0=最低压力，100=最高压力） |

## 两者联动关系

- **充电时**：压缩热存入 HTV（热 SOC↑），CO₂ 被压缩存入 HPT（气 SOC↑）→ **两者同向上升**
- **放电时**：回热从 HTV 取热（热 SOC↓），CO₂ 从 HPT 释放（气 SOC↓）→ **两者同向下降**
- 热量与压力通过无量纲参数 β1、β2、α1、α2 耦合：压缩/膨胀功率 → 同时影响热存储和气存储

**关键观察**：
- 凌晨充电（0-8h）→ 双 SOC 同步上升
- 午间放电（12-15h）→ 双 SOC 同步下降
- 晚间再充电（22-24h）→ 双 SOC 再次上升
- 热 SOC 和气 SOC 的趋势高度正相关，验证了双 SOC 耦合模型

# Fig09 — Heat-dependent vs Power-dependent 调度模式

Fig09 对应论文 Fig.7，展示当调度指令超出原始可行域时，两种**降级运行策略**。

## 背景

放电工况下约束线 `Q = −β2·W + Q₀` 为负斜率。当用户同时要求高发电 + 高供热，指令点可能落在灰色可行域**之外**。系统只能降级运行。

## 三种区域

| 颜色 | 含义 |
|------|------|
| 灰色 | 原始可行域——储热充裕时的全部可运行范围 |
| 红色 | **Heat-dependent**：优先保证供热 8 MW → 缩减发电上限（10→~7.8 MW） |
| 绿色 | **Power-dependent**：优先保证发电 8 MW → 缩减供热上限（9.5→~7.7 MW） |

## 两个标记点

- 红点（Heat-dep.）：固定 Q=8 MW，最大只能发 ~7.8 MW
- 绿点（Power-dep.）：固定 W=8 MW，最大只能供热 ~7.7 MW

**应用意义**：实际调度中根据用户优先级选择模式——医院/居民区优先供热（Heat-dependent），工业园区优先供电（Power-dependent）。

# Fig10 — 热供应率 X 随透平入口温度变化

Fig10 展示论文 Section 4.2.1 的可变工况分析：改变透平入口温度对系统**热电分配比例**的影响。

## 双 y 轴

| 曲线 | y 轴 | 含义 |
|------|------|------|
| 蓝色（X %） | 左轴 | 热供应率：储热中用于直接供热的比例 |
| 红色（β₂） | 右轴 | 修正后的回热系数 |

## 物理逻辑

- 透平入口温度 **升高** → 每单位 CO₂ 做功能力增强 → 同样发电量需要的回热减少 → β₂ 下降 → 更多储热可用于供热 → **X 上升**
- 透平入口温度 **降低** → CO₂ 做功能力减弱 → 需要更多回热 → β₂ 上升 → 供热减少 → **X 下降**

通过调节透平入口温度，可以在电效率和供热效率之间进行灵活调度。

## 6 条彩色多边形

每条对应一个时刻（h1→h8→h12→h15→h20→h23），标签沿右边界垂直错开避免重叠。

**核心对比**：
- **Scenario B 的域总是 ≤ Scenario A**：电负荷越重 → 放电越多 → 储热消耗越快 → 约束越紧
- 中午-下午（h12→h15）域收缩最明显：此时放电功率最大，储热快速消耗
- 这一对比验证了论文的核心结论：适度增加电负荷调度可以延长 CCES-CHP 放电时间，提高运行经济性
