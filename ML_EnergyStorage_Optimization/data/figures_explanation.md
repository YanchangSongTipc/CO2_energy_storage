### ML 优化的三个核心策略

#### (1) 降低单次放电功率，换持续时间

储罐能量 = 功率 × 时间。规则调度倾向于满功率放电（10 MW），ML 优化根据 SOC 状态自适应调节——储热紧张时自动降功率：

```
规则: [10, 10, 10, 10, 10, 0, 0, 0, 0]  → 5h 满功率, 然后枯竭
ML:   [8, 7, 6, 5, 5, 4, 3, 3, 2]     → 9h 递减功率, 覆盖全天
```

#### (2) 电热比例的动态调配

放电时 `Q_HTV 消耗 = β2·W_T + Q_heat`。规则调度固定分配，ML 在每个时刻求解最优 (W, Q) 组合——电价高时多发少供，热价高时多供少发，让每单位储热产生最大价值。

#### (3) 利用充电间隙"回血"

24h 调度中有多个充电窗口（凌晨风电过剩、午间低谷），ML 优化精确计算充电量——充到刚好够下一段放电用，不浪费储能空间。

### 物理上限

储罐总能量 = `Q_HTV_upper = 150 MWh`，满功率放电消耗率 = `β2 × 10 + Q_heat ≈ 23 + 8 = 31 MW`。所以**连续满功率放电最多 ~4.8 小时**。ML 优化没有改变这个物理上限，而是通过降功率和错峰调度把有限的储能覆盖了更长的时段。

### 性能对比

| 指标 | 规则调度 | ML 优化 |
|------|---------|--------|
| 总充电量 | 66.0 MWh | 66.0 MWh |
| 总放电量 | 48.3 MWh | 46.1 MWh |
| 总供热量 | 43.0 MWh | **46.3 MWh** |
| 放电时长 | 15 h | **24 h** |
| 末态 SOC | 0.0% | **5.0%** |

## 2. feasibility_domain_change.png — ML 优化前后的可行域变化

### 图中元素

| 元素 | 含义 |
|------|------|
| 灰色区域 | 基准可行域（T_turb=483K, β2=2.31） |
| 蓝色区域 | ML 优化后可行域（T_turb=503K, β2=1.78） |
| 青色区域 | ML 优化**扩展**的区域 |
| 黑色实线 | 基准约束边界 |
| 蓝色实线 | 优化后约束边界（斜率更缓） |
| 黑色圆点 | 基准 W_max（零供热时最大发电） |
| 蓝色圆点 | 优化后 W_max |

四个子图对应 SOC_heat = 20%, 40%, 60%, 80%。

### 核心变化：β2 降低 → 约束线斜率变缓

ML 优化通过调节透平入口温度改变了系统参数：

```
T_turb:  483K → 503K
β2:      2.31 → 1.78  (−23%)
```

约束方程 `Q = −β2·W + Q₀`：

- β2 越小 → 斜率越平缓 → 同样的储热下可支持更大的发电功率
- 零供热最大功率: `W_max = Q₀ / β2` → β2 降低 23% → W_max 增加 ~30%

### 定量分析

| SOC | 基准面积 | 优化面积 | 增幅 | 基准W_max | 优化W_max |
|-----|---------|---------|------|----------|----------|
| 10% | 30.8 | 40.0 | **+29.8%** | 6.5 MW | 8.4 MW |
| ≥20% | 47.5 | 47.5 | 0% | 10 MW | 10 MW |

**低 SOC 时可行域显著扩展**（+29.8%），高 SOC 时受设备上限（W_max=10 MW, Q_max=9.5 MW）约束，面积不再变化。

### 放电时长对比

| 场景 | 满功率回热消耗 | 放电时长 |
|------|--------------|---------|
| 基准 (483K) | β2×10 = 23.1 MW | ~6.5 h |
| ML优化 (503K) | β2×10 = 17.8 MW | **~8.4 h** |
| 节省 | 5.3 MW | **+1.9 h** |

### 物理本质

```
提高 T_turb → CO₂做功能力↑ → 单位发电需要的回热↓ → β2↓
→ 约束线 Q = −β2·W + Q₀ 斜率变缓
→ 同样储热下, W_max 从 Q₀/β2_old 增大到 Q₀/β2_new
→ 可行域向右扩展, 放电时长延长
```

这就是摘要中"放电时长从 5h 延长到 8h"的物理机制——**不是增加了储罐容量，而是通过 ML 优化调节运行参数，让每单位储热发挥更大效用**。

## 3. 两个关键问题的回答

### Q1: 储罐容量有限，如何实现连续放电？

**答**: 三个策略叠加，不是突破物理上限。

1. **降功率换时间**: 储罐能量 = 功率 × 时间。ML 优化在储热紧张时自动降低放电功率，用时间换覆盖——满功率 10 MW 只能放 4.8h，但降到 5 MW 可以放 9.6h。

2. **电热动态调配**: 放电消耗 = β2·W + Q_heat。ML 在每个时刻动态分配——电价高时多发少供，热价高时多供少发。

3. **利用充电间隙回血**: 精确计算每次充电量，充到刚好够下一段放电。

物理上限：满功率满供热连续放电最多 ~4.8h。24h 调度中的"连续放电"实际上是**多个降功率放电段的拼接**。

### Q2: 经过机器学习优化后的可行域有什么变化？

**答**: 变化体现在两方面。

**第一，ML 本身不改变可行域**。当前实现中 ML 学到的是与物理约束完全相同的边界（准确率 100%），因为约束是简单的线性不等式。ML 的价值在于预测速度——比运行完整热力学仿真快数百倍，适合实时调度。

**第二，ML 优化的真正价值是改变了运行参数**。摘要的核心思路是：ML 模型学习不同透平入口温度下的 β2 变化规律，然后在调度优化中选择最优 T_turb：
- T_turb ↑ → β2 ↓ → 约束线斜率变缓 → 可行域扩展 → 放电时长 5h→8h
- 这就是 `feasibility_domain_change.png` 展示的核心效果：低 SOC 时可行域面积 +29.8%
